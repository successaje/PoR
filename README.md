# Proof-of-Reality (PoR) Protocol
> **Decentralized AI consensus for verifying real-world truth.**

Proof-of-Reality (PoR) is an institutional-grade intelligence protocol built for the Mantle Network. It utilizes a multi-agent AI debate system to verify off-chain, real-world assets (starting with Real Estate) and mints cryptographic "Truth Certificates" on-chain that dynamically reflect the consensus score.

---

## 🏛️ System Architecture

The PoR architecture is designed as a mission-critical pipeline resembling a Palantir or Bloomberg intelligence terminal.

### 1. The Verification Pipeline
1. **Data Ingestion:** A user inputs a registry identifier (e.g., `REG-8492-TX`) for a real-world asset.
2. **Node Activation (AI Orchestration):** Specialized AI Agents are spun up. Each agent represents a different domain of truth:
   - **Geo-Spatial Agent:** Analyzes satellite imagery and physical boundaries.
   - **Financial Agent:** Cross-references tax records, mortgages, and valuations.
   - **Legal/Title Agent:** Verifies public deeds and ownership history.
3. **The Debate Chamber:** Agents do not just output a result; they *debate*. If the Geo-Spatial agent finds a boundary anomaly but the Legal agent confirms the deed, they debate until a mathematical consensus is reached.
4. **On-Chain Certification:** Upon reaching a high-confidence consensus, the protocol issues a `TruthCertificateNFT` on the Mantle Network, permanently recording the asset's verified state.

### 2. Smart Contract Design (Mantle Network)
The core of the on-chain infrastructure is the `TruthCertificateNFT.sol` contract (ERC721).
- **Minting Truth:** The contract records the `assetId`, `consensusScore`, `verificationTimestamp`, and the cryptographic `evidenceHash` generated by the AI debate.
- **Dynamic Truth Decay (Simulated ERC-8004):** Truth is not static. Real estate boundaries change, and valuations shift. The smart contract includes a `decayTimer`. As time passes without a re-verification, the `consensusScore` decays by 5% per epoch. This enforces continuous data freshness and oracle reliance.

## 🛠️ Technology Stack

**Frontend (Mission Control UI)**
- **Framework:** Next.js 16 (App Router), React 19
- **Styling:** TailwindCSS v4 (Strict Matte-Black Institutional Design System)
- **Web3 Integration:** Wagmi v2, Viem, RainbowKit (Custom `ConnectButton` overriding default UI)
- **Motion:** Framer Motion (Restrained, procedural animations)

**Smart Contracts (On-Chain Truth)**
- **Framework:** Foundry
- **Language:** Solidity `^0.8.24` (Utilizing Cancun `mcopy` opcodes)
- **Libraries:** OpenZeppelin v5
- **Network:** Mantle Sepolia Testnet

**Backend (The Agent Engine - WIP)**
- **Framework:** FastAPI (Python)
- **AI Orchestration:** LangGraph / LangChain

The current repository contains the **V2 Institutional UI** and the fully integrated **Smart Contracts**.

## 💻 Local Development

### Prerequisites
- Node.js `v20+` & `pnpm`
- Foundry (for smart contracts)

### Running the Frontend
```bash
cd apps/web
pnpm install
pnpm run dev
```
Navigate to `http://localhost:3000`.

### Deploying the Contracts
```bash
cd contracts
# Create a .env file with your PRIVATE_KEY (Must start with 0x)
forge script script/Deploy.s.sol:DeployScript --rpc-url https://mantle-sepolia.drpc.org --broadcast
```
Copy the deployed address and update `apps/web/src/app/verify/page.tsx`.
